--------------------------------------------------------------------------------

3. GCPs well distributed (corners / across image)
4. Low residual / RMS error
5. Correct CRS (EPSG:5234 Kandawala)
6. Digitization of features (e.g., building footprint)
7. Attribute table with required fields
8. Area calculation
9. Map with standard elements
10. Critical discussion of geographic location
11. Screenshots in appendix
12. Citations and referencing

--------------------------------------------------------------------------------
GCP (GROUND CONTROL POINTS) – 4 DOTS
--------------------------------------------------------------------------------

✓ 4 GCPs is CORRECT and SUFFICIENT

- Minimum for Polynomial 1 transformation: 3 points
- 4 points recommended for better accuracy and redundancy
- Sample assignment used exactly 4 GCPs for MOHE digitization

GCP Placement Tips:
- Spread across the image: corners or corners + center
- Use identifiable features (road intersections, building corners)
- Source (image) ↔ Destination (map coordinates) must match same real-world location

Your GCP Table (dX=0, dY=0, Residual=0):
- Zero residual = perfect fit at those 4 points
- This is EXCELLENT for georeferencing accuracy
- Proceed to "Start Georeferencing" / Apply

--------------------------------------------------------------------------------
PART 1: GEOREFERENCING
--------------------------------------------------------------------------------

Step 1.1  Open QGIS → Raster → Georeferencer

Step 1.2  Open image: SL_Areal_Image_2025.jpg

Step 1.3  Set Transformation Settings
          - Settings → Transformation Settings
          - Transformation type: Polynomial 1 (or Polynomial 2 for 6+ GCPs)
          - Resampling: Nearest neighbour
          - Output CRS: EPSG:5234 (Kandawala / Sri Lanka Grid)
          - Output file: georeferenced_image.tif
          - OK

Step 1.4  Add 4 GCPs
          - Click "Add Point" (or click on image)
          - Click on image (source) → Enter map coordinates (destination)
          - Get coordinates from: Google Earth (convert to EPSG:5234) or
            existing georeferenced layer in QGIS
          - Place at 4 corners or spread across image

Step 1.5  Check GCP Table
          - dX, dY, Residual should be low (0 = perfect)
          - Your 4 dots with Residual 0 = EXCELLENT ✓

Step 1.6  Start Georeferencing
          - Click "Play" / "Start Georeferencing"
          - Georeferenced raster will load in QGIS

Step 1.7  Screenshot: Georeferencer with 4 GCPs
          Save as: Task_C screenshot/step1_georeferencer.png

--------------------------------------------------------------------------------
PART 2: DIGITIZATION
--------------------------------------------------------------------------------

Step 2.1  Create new shapefile
          - Layer → Create Layer → New Shapefile Layer
          - Type: Polygon
          - CRS: EPSG:5234
          - Add fields: id (Integer), name (String), type (String), area_sqm (Real)
          - Save: MOHE_buildings.shp (or appropriate name)

Step 2.2  Toggle editing on the new layer

Step 2.3  Digitize building/feature
          - Add Polygon Feature tool
          - Trace the building perimeter on georeferenced image
          - Follow visible edges (roads, building outlines)
          - Save edits

Step 2.4  Calculate area
          - Open attribute table → Field calculator
          - area_sqm = $area
          - OK

Step 2.5  Screenshot: Digitized feature on image
          Save as: Task_C screenshot/step2_digitization.png

--------------------------------------------------------------------------------
PART 3: MAP AND REPORT
--------------------------------------------------------------------------------

Step 3.1  Add base map (OpenStreetMap)
Step 3.2  Print Layout: Title, Legend, North Arrow, Scale Bar
Step 3.3  Export final map
Step 3.4  Critical discussion: Geographic location, significance, area calculated

--------------------------------------------------------------------------------
COORDINATE SOURCES FOR GCPs
--------------------------------------------------------------------------------

If you need destination coordinates for GCPs:

Option A – Google Earth:
- Find location, note Lat/Long (WGS84)
- Convert to EPSG:5234 using QGIS or online converter

Option B – Transformation Parameters (Question-(d)):
- False Easting: 500000 (or 200000 for 1:10,000/1:50,000)
- False Northing: 500000 (or 200000)
- Central Meridian: 80°46'18.167"E
- Central Parallel: 7°00'1.697"N

Option C – Use existing georeferenced layer:
- Add Boundary.shp or other layer in EPSG:5234
- Click on map to get coordinates for GCP destination
